The Reader

About These Readings

I read with multiple decks because each one shows a different layer of what’s happening. The Sibilla maps events and people with a directness that other systems avoid. Kipper tracks movement — who’s acting, who’s nearby, what’s coming toward you. Tarot speaks to the inner experience. Oracle names the archetypes and energies operating beneath everything else.

I walk you through the reading rather than handing you a list. Every card gets context, every position gets its lens explained, and the synthesis at the end ties the whole picture together. The goal is that you leave understanding not just what the cards said, but how to think about what they’re showing you.

Readings are delivered as private, mobile-optimized pages — built to be read at your own pace, saved as a PDF, and returned to whenever the week asks you to check back in.

How It’s Organized

Repo & File Structure

Repo Naming Convention
  • Ongoing clients — one repo that holds all readings:
    [Name]-Readings → e.g., Natalie-Citadel-Readings, Naomi-Citadel-Readings
  • Week ahead readings — when a client gets their own dedicated repo per week:
    [Name]-Week-Ahead-[Month]-[Day]-[Year] → e.g., Armani-Week-Ahead-April-6-2026
  • One-time readings — soul portraits, annual, special:
    [Name]-[Type]-[Year] → e.g., Tiffany-s-Soul-Portrait, Dionna-s-Soul-Portrait
  • Main hub — this page, always at root:
    jaylon-jackson.github.io
Within an Ongoing Client Repo
index.html   ← always the current/latest reading
deck-opener.jpg   ← static deck photo, reused each week
face-down.jpg   ← current week spread (face down)
face-up.jpg   ← current week spread (revealed)
archive/
  week-march-30-2026.html
  monthly-march-2026.html
  ...
Weekly Workflow — New Reading Arrives
  • Move current index.html → archive/week-[date].html
  • Replace face-down.jpg and face-up.jpg with this week’s spread photos
  • New index.html goes in root — client URL stays the same, no new link to send
  • deck-opener.jpg stays permanently; never replaced unless decks change
  • Update this hub’s directory with the new date range and status
HTML Standards (Every Reading Page)
  • Images as separate repo files — never base64 in HTML (keeps HTML under 80 KB)
  • All special characters as HTML entities — &mdash; &middot; &rsquo; etc. (Safari-safe)
  • Countdown timer + save instructions near bottom of every reading
  • Expiry gate overlay — week readings expire Sunday midnight, monthly readings expire 40 days out
  • Print CSS hides timer/save section — PDF stays clean
  • Deck opener image: static base64 (one-time embed, small, never changes)

What’s Available

Reading Types

  • Week Ahead — Seven positions covering theme, supports, friction, surprise, challenge, advice, and synthesis. Sibilla as the primary deck. Citadel Oracle for energetic context. Key influence card drawn from the bottom of the deck. Delivered Monday, expires Sunday midnight.
  • Monthly — Full multi-deck reading using Sibilla, Kipper, Tarot, and Oracle. Covers the external terrain, internal experience, key energies, and a synthesis tying all four layers together. Delivered at the start of the month, expires 40 days out.
  • Soul Portrait — A deep-dive one-time reading. No expiry. Delivered as a permanent page the recipient keeps.
  • Annual — Full-year overview using multiple decks. One card or cluster per month, with an overarching theme card for the year. Delivered January or at the start of a personal new year.
